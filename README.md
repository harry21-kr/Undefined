## 브랜치 전략

### *main branch*

개발이 완료된 후 배포되는 브랜치입니다.

### *develop branch*

개발 단계에서 git-flow의 master 역할을 하는 브랜치입니다.

### *feature branch*

새로운 기능이나 레이아웃을 추가하는 브랜치입니다.

### *fix branch*

개발되었던 기능이나 레이아웃을 수정하는 브랜치입니다.

### *refactor branch*

개발되었던 코드를 리팩토링하는 브랜치입니다.
